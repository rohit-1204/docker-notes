# 📖 What is Docker Logging?

Docker Logging is the process of collecting and storing logs generated by containers.

Logs help you:

- 🔍 Troubleshoot applications
- 📊 Monitor container activity
- 🚨 Detect errors
- 📜 Audit application behavior

---

# 💡 Why Logging is Important?

Without Logging:

```text
Application Error

↓

No Logs

↓

Cannot Find Problem ❌
```

With Logging:

```text
Application Error

↓

Docker Logs

↓

Identify Issue

↓

Fix Problem ✅
```

Benefits:

- ✅ Troubleshooting
- ✅ Monitoring
- ✅ Debugging
- ✅ Performance Analysis
- ✅ Security Auditing

# 📦 Docker Logging Drivers

Docker supports multiple logging drivers.

| Driver | Description |
|---------|-------------|
| json-file | Default logging driver |
| local | Optimized local storage |
| syslog | Send logs to Syslog |
| journald | Linux systemd logging |
| fluentd | Forward logs to Fluentd |
| gelf | Graylog logging |
| awslogs | Send logs to Amazon CloudWatch |
| splunk | Send logs to Splunk |

View logs:


```bash

```

Monitor logs in real time:

```bash

```

Display the last 20 log lines:


---

# 🕒 Show Logs with Timestamps

```bash

```

---

# ⚙️ Configure Logging Driver

Run a container with the **local** logging driver:

```bash
docker run \
--log-driver local \
nginx
```

Use **json-file**:

```bash
docker run \
--log-driver json-file \
nginx
```

# ☁️ Centralized Logging

In production, logs are usually sent to centralized logging platforms.

```text
Containers
      │
      ▼
Docker Logging Driver
      │
      ▼
CloudWatch / ELK / Splunk
      │
      ▼
Dashboard & Alerts
```

Popular platforms:

- Amazon CloudWatch
- ELK Stack
- Grafana Loki
- Splunk
- Fluentd

---

# 🔄 Log Rotation

Log rotation prevents log files from growing indefinitely.

Example:

```bash
docker run \
--log-opt max-size=10m \
--log-opt max-file=5 \
nginx
```

This configuration:

- Maximum log size: **10 MB**
- Keep only **5 log files**

# 🏆 Best Practices

- ✅ Use centralized logging in production
- ✅ Configure log rotation
- ✅ Avoid storing unlimited logs
- ✅ Monitor application logs regularly
- ✅ Use structured log formats (JSON)
- ✅ Send logs to monitoring platforms
- ✅ Secure access to log files

| Use Case | Logging Solution |
|----------|------------------|
| Local Development | json-file |
| Production Servers | local |
| AWS Deployment | awslogs |
| ELK Stack | fluentd |
| Enterprise Monitoring | Splunk |
| Linux Systems | journald |

# 📝 Key Takeaways

- Docker captures logs from `stdout` and `stderr`.
- The default logging driver is `json-file`.
- Log rotation prevents excessive disk usage.
- Centralized logging simplifies monitoring.
- Production environments should integrate with log management platforms.
